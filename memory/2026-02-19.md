## 04:30 UTC - Experimento Q&A con Depth=2

### Lo que intentamos:
- Sistema multiagente con orquestador y 2 subagentes conversando
- Yo lanzo orquestador, orquestador lanza GLM-5 + Gemini

### Lo que descubrimos:

| Problema | Causa | Solución |
|----------|-------|----------|
| GLM-5 rate limit | Solo **1 request concurrente** | No puedo spawnear GLM-5 si yo lo uso |
| Gemini agotado | Límite **20 requests/día** | Reset automático, pero limitado |
| Depth=2 no funciona | Requiere modelos diferentes para cada nivel | Usar GLM-5 en depth=0, Gemini en depth=1 |

### Artefacto actualizado:
- `memory/spec-qa-orchestrator.md` — Añadida sección "LIMITACIONES REALES"

### Conclusión:
**Orquestación concurrente con depth=2 NO es posible en tier gratuito.**

Soluciones viables:
1. MAIN hace de orquestador → spawn UN subagente con modelo diferente
2. Orquestación secuencial → esperar turns
3. Añadir créditos a Claude ($5) → fallback real
4. Diseñar sin depth=2 → preguntas directas
